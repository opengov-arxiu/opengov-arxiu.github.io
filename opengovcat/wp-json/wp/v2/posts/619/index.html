El sism\u00f3grafo<\/a> es s\u00f3lo un comienzo (explicaciones<\/a>). Creemos que con base en los datos que tenemos, se pueden construir servicios y visualizaciones mucho m\u00e1s lindas y m\u00e1s \u00fatiles.<\/p>\n

Aqu\u00ed juntamos algunas de las ideas acerca de las posibilidades que tenemos con el sism\u00f3grafo.<\/p>\n

    \n
  • Construir un servicio de alerta “20 cambios en el Depto. de Justicia”<\/li>\n
  • Constriuir un robot-periodista que publica los cambios todos los d\u00edas en un blog \u00a0y \/ o en Twitter.<\/li>\n
  • Construir una curva de frecuencia\u00a0de cambios, parecida a un sism\u00f3grafo real<\/li>\n
  • Buscar las \u00e1reas m\u00e1s abandonadas<\/li>\n
  • Buscar las personas que re\u00fanen el mayor n\u00famero de responsabilidades<\/li>\n<\/ul>\n

    P\u00f3nganse en contacto con @opengovcat<\/a> si quieren construir algo. Abajo ya pueden leer\u00a0algunas de las cosas\u00a0que tienen que saber de la API.<\/div>\n

    La\u00a0API del sism\u00f3grafo<\/h2>\n

    Por ahora, la API tiene un s\u00f3lo\u00a0endpoint:\u00a0<\/em><\/h4>\n
    http:\/\/18.184.0.9\/sismograf\/api\/canvis\/<departamento>\/<comienzo>\/<fin>\r\n<\/pre>\n
    Por ejemplo:<\/p>\n
    http:\/\/18.184.0.9\/sismograf\/api\/canvis\/all\/2014-01-01\/2014-01-31<\/a>\r\n<\/pre>\n
     <\/p>\n
    El resultado<\/h4>\n
    \u00a0{ status : \"ok\",\r\n   resultados : { borados    : [ (... lista de entidades borradas ...) ],\r\n                  canvisnom  : [ (... lista de entidades con nombres cambiados ...) ],          \r\n\r\n                  canvisresp : [ (... lista de entidades con responsabilidades cambiadas ...) ],\r\n\r\n                  nuevos     : [ (... lista de entidades nuevas ...) ]\r\n                 }\r\n }\r\n\r\n<\/pre>\n
    Propiedades de las entidades:<\/p>\n
      \n
    • “centres”: Lista de centros para entidades que tienen varias sedes, valor copiado de la fuente.<\/li>\n
    • “cid”:\u00a0\u00a0ID interno del cambio. Con la URL http:\/\/18.184.0.9\/sismograf\/c\/<id> se puede obtener una representaci\u00f3n HTML.<\/li>\n
    • “class”:\u00a0\u00a0Clase CSS del cambio<\/li>\n
    • “id”: “15377”:\u00a0ID de la entidad. Con la URL\u00a0http:\/\/www14.gencat.cat\/sacgencat\/AppJava\/organisme_fitxa.jsp?codi=<id> se puede obtener la ficha de la entidad<\/li>\n
    • “iddep”: “5823”: ID de la entidad superior. Con la URL\u00a0http:\/\/www14.gencat.cat\/sacgencat\/AppJava\/organisme_fitxa.jsp?codi=<id> se puede obtener la ficha de la entidad<\/li>\n
    • “nom”: Nombre seg\u00fan fuente<\/li>\n
    • “oldnom” : Nombre anterior<\/li>\n
    • “oldresp”: Responsable anterior<\/li>\n
    • “oldstamp”: Fecha cuando fue registrado por primera vez el responsable o el nombre anterior<\/li>\n
    • “resp”: Responsable actual<\/li>\n
    • “stamp”: Fecha del cambio<\/li>\n
    • “test” : ver abajo.<\/li>\n<\/ul>\n
      Las entidades cambiadas tienen una propiedad\u00a0“<\/span>test<\/strong>“, un n\u00famero entre 0 y 2, und valor mayor significa que en el nombre (canvisnom) o en el responsable (canvisresp) los cambios fueron mayores. Algunos cambios de responsables o nombres son simples correciones \u00a0– con la propiedad “test”\u00a0es posible diferenciar. Los cambios con “test”<0.1 son filtrados por la\u00a0API y no aparecen en el resultado. “test” se calcula as\u00ed: <\/span>Leveshtein edit distance(old,new) \/ Max(len(old),len(new)).<\/em><\/p>\n
