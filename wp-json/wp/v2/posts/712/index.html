The seismograph<\/a>\u00a0(sism\u00f2graf in Catalan)\u00a0\u00a0is just the beginning (explanation<\/a>). We believe we can build much more useful services and prettier data visualizations based on this data .<\/p>\n

Here are some ideas about what we think we could do with this seismograph:<\/p>\n

    \n
  • Build an alert service by subject area, for example\u00a0“20 most recent changes in the Dept. of Justice”<\/li>\n
  • Build a journalist-robot that publishes daily changes on a blog and\/or on\u00a0Twitter.<\/li>\n
  • Build a change frequency chart, similar to a real\u00a0seismometer<\/li>\n
  • Look for the most neglected areas<\/li>\n
  • Look for people who are responsible for a few entities<\/li>\n<\/ul>\n

    Please contact\u00a0@opengovcat<\/a>\u00a0if you are interested in building any of the above. Below you will find some details to know about the API.<\/strong><\/p>\n

    The seismograph\u00a0API<\/h2>\n

    So far, the API has only one\u00a0endpoint:\u00a0<\/em><\/h4>\n
    http:\/\/18.184.0.9\/sismograf\/api\/canvis\/<departamento>\/<comienzo>\/<fin>\r\n<\/pre>\n
    For example:<\/p>\n
    http:\/\/18.184.0.9\/sismograf\/api\/canvis\/all\/2014-01-01\/2014-01-31<\/a>\r\n<\/pre>\n
     <\/p>\n
    The result<\/h4>\n
    \u00a0{ status : \"ok\",\r\n   resultados : { borados    : [ (... list of erased entities ...) ],\r\n                  canvisnom  : [ (... list of entities with new names ...) ],          \r\n\r\n                  canvisresp : [ (... list of entities with new responsibilities ...) ],\r\n\r\n                  nuevos     : [ (... list of new entities ...) ]\r\n                 }\r\n }\r\n\r\n<\/pre>\n
    Properties:<\/p>\n
      \n
    • “centres”: List of centres for entities with several headquarters, value copied from the source.<\/li>\n
    • “cid”: \u00a0internal ID of the change. On the\u00a0URL http:\/\/18.184.0.9\/sismograf\/c\/<id> you can obtain an HTML representation.<\/li>\n
    • “class”: \u00a0 CSS \u00a0change class<\/li>\n
    • “id”: “15377”:\u00a0ID of the entity. On the\u00a0URL\u00a0http:\/\/www14.gencat.cat\/sacgencat\/AppJava\/organisme_fitxa.jsp?codi=<id> you can obtain the entity index card<\/li>\n
    • “iddep”: “5823”: ID of the higher entity. On the URL\u00a0http:\/\/www14.gencat.cat\/sacgencat\/AppJava\/organisme_fitxa.jsp?codi=<id> you can obtain the entity index card<\/li>\n
    • “nom”: Name according to the source<\/li>\n
    • “oldnom” :\u00a0Previous name<\/li>\n
    • “oldresp”:\u00a0Previous person responsible<\/li>\n
    • “oldstamp”: Date when a person responsible or a previous name was registered for the first time<\/li>\n
    • “resp”: Present person responsible<\/li>\n
    • “stamp”:\u00a0Date of change<\/li>\n
    • “test” : see below.<\/li>\n<\/ul>\n
      The entities that have changed have a “<\/span>test<\/strong>” property, a number between 0 and 2. A value higher than that means that there were more changes in it than the name (canvisnom) or the person responsible (canvisresp). Some such changes are simple spelling corrections and this\u00a0“test” property makes them easier to identify. The changes with\u00a0“test”<0.1 are filtered by the\u00a0API and don’t show in the results. “test” is calculated as follows: <\/span>Leveshtein edit distance(old,new) \/ Max(len(old),len(new)).<\/em><\/p>\n
